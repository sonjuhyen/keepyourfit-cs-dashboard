# 킵유어핏 CS 대시보드 🦦

채널톡 CS 자동화 시스템의 승인율을 추적하고, 승인된 답변을 DB화하는 대시보드입니다.

## 🎯 주요 기능

### 📊 CS 현황 대시보드
- 오늘/이번주/이번달 문의 수 추적
- 자동처리 vs 수동처리 비율 분석
- 승인율 모니터링 (승인/거부/대기)
- 문의 유형별 통계 (배송, 환불, 제품, 일반)
- 평균 응답 시간 표시
- 실시간 차트로 데이터 시각화

### 💬 답변 관리
- 승인된 답변 목록 관리
- 답변 검색 및 카테고리별 필터링
- 답변별 키워드 태그 시스템
- 답변 수정/삭제 기능
- 새 답변 등록 및 편집

### 📋 승인 로그
- 모든 문의 + 답변 초안 + 승인/거부 기록
- 거부 사유 상세 기록
- 시간순 정렬 및 필터링
- 날짜별, 상태별 검색 기능

### ⚙️ 자동처리 규칙
- 승인된 답변 기반 자동처리 규칙 관리
- 키워드 매칭 규칙 편집
- 자동처리 ON/OFF 토글
- 규칙별 매칭 횟수 추적

## 🛠️ 기술 스택

- **프론트엔드**: HTML5 + Tailwind CSS + Alpine.js
- **데이터 저장**: localStorage (클라이언트 사이드)
- **차트**: Chart.js
- **배포**: GitHub Pages 호환 (정적 파일)
- **언어**: 한국어

## 📁 파일 구조

```
keepyourfit-cs-dashboard/
├── index.html          # 메인 대시보드 (SPA)
├── css/
│   └── style.css       # 커스텀 스타일
├── js/
│   ├── app.js          # 메인 애플리케이션 로직 (Alpine.js)
│   ├── data.js         # 데이터 관리 (localStorage)
│   └── charts.js       # 차트 관리 (Chart.js)
└── README.md           # 프로젝트 설명
```

## 🚀 사용법

### 로컬 실행
1. 프로젝트 폴더를 다운로드합니다
2. `index.html` 파일을 브라우저에서 열거나 로컬 서버로 실행합니다
3. 샘플 데이터가 자동으로 로드됩니다

### GitHub Pages 배포
1. GitHub 저장소에 파일들을 업로드합니다
2. Settings > Pages에서 소스를 설정합니다
3. `https://계정명.github.io/저장소명/`으로 접속합니다

## 📊 샘플 데이터

초기 실행 시 다음 샘플 데이터가 포함됩니다:

**처리된 CS 케이스:**
- ✅ 섭취법 문의 2건 (승인됨)
- ✅ 제품 용량 문의 1건 (승인됨)  
- ⏸️ 주문번호 배송 조회 1건 (에스컬레이션)
- 🚫 스팸 5건 (자동 차단)

**자동처리 규칙:**
- 섭취법 문의 자동 응답
- 용량 문의 자동 응답
- 스팸 자동 차단

## 💾 데이터 저장

- 모든 데이터는 브라우저 localStorage에 저장됩니다
- 서버가 필요 없는 완전 클라이언트 사이드 애플리케이션
- 데이터 백업은 브라우저 개발자 도구에서 export/import 가능
